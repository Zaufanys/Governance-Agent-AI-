      Send traces (ingestion API)
      
      
      Agents and pipelines post traces to POST /api/ingest with an API-key header. Each trace is scored by the governance engine and appears here immediately.
      
      
      curl -X POST http://localhost:4175/api/ingest \
  -H "Authorization: Bearer <your-api-key>" \
  -H "Content-Type: application/json" \
  -d '{
    "agent": "RFQ Copilot",
    "input": "Summarize RFQ-2026-PS-114",
    "tools": [{ "name": "search_rfq_docs", "type": "read" }],
    "approvalRequired": true,
    "approved": false,
    "groundedness": 0.9
  }'
      
    
      

    
    
      
      
      Governance principles
      
      
      
        
      
          Least privilege
          Agents should only reach the tools and data a task actually needs — read vs. write matters.
        
      
        
      
          Human approval
          Write actions, pricing, and customer-facing output pass through a human before they take effect.
        
      
        
      
          Trace & evaluate
          Every model call, retrieval, tool call, and output is captured, scored, and auditable.
