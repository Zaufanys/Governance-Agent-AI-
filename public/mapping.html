From prototype to production observability

This prototype uses local JSON and a deterministic policy engine so the governance concepts are easy to read. In production the same concepts map onto managed tracing, evaluation, and tool-governance platforms. The table below is a conceptual mapping, not a product endorsement or integration guide.

This page is a conceptual mapping for orientation. No vendor connections or credentials are configured by the service itself.

Azure AI Foundry

Managed agent runtime with built-in tracing/observability (via Application Insights & OpenTelemetry), evaluations for groundedness and safety, and Prompt Shields for injection defence.

Databricks MLflow 3

GenAI tracing with spans for model and tool calls, LLM-as-judge evaluation, feedback capture, and production quality monitoring on the lakehouse.

OpenTelemetry

Vendor-neutral standard. GenAI semantic conventions describe spans for model calls, tool calls, and token usage — exportable to any backend as traces, metrics, and logs.

MCP & tool governance

The Model Context Protocol standardises how agents reach tools. Governance lives at the boundary: which tools are exposed, read vs. write scope, allowlists, and approvals.
